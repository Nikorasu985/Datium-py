================================================================================
GUÍA MAESTRA DE USUARIO - DATIUM (SISTEMA INTEGRAL NO-CODE AI)
================================================================================

Datium es una solución empresarial diseñada para democratizar la gestión de bases
de datos y la inteligencia artificial, eliminando la barrera del código.

--------------------------------------------------------------------------------
1. MÓDULO DE SEGURIDAD Y PERMISOS (SERVICES & SECURITY)
--------------------------------------------------------------------------------

Datium implementa un modelo de seguridad multinivel para proteger tus datos:

1.1. MODOS DE SEGURIDAD DEL SISTEMA:
   - Abierto (None): Acceso total para el propietario del sistema.
   - Seguridad General (General): Requiere una "General Password" para cualquier
     operación dentro del sistema. Ideal para entornos compartidos básicos.
   - Seguridad Individual (Individual): Basado en invitaciones (In development).
     Permite auditoría específica por usuario.

1.2. AUDITORÍA INTEGRAL:
   - Registro de Actividad (Audit Log): Captura Acción, Detalles, IP, Usuario y
     Fecha. Es posible filtrar por rango de fechas y términos de búsqueda.
   - Auditoría de Seguridad: Registra eventos críticos clasificados por severidad:
     * Low: Cambios menores en la configuración.
     * Medium: Intentos de acceso fallidos.
     * High: Eliminación de tablas o cambios en modos de seguridad.

--------------------------------------------------------------------------------
2. MÓDULO BUILDER: CONSTRUCCIÓN DE ESTRUCTURAS
--------------------------------------------------------------------------------

Este módulo permite definir "Sistemas" (Bases de Datos) y "Tablas".

2.1. PARÁMETROS DE CAMPO (ESPECIFICACIONES TÉCNICAS):
   - Texto: Alfanumérico simple.
   - Número: Valores enteros o decimales para cálculos.
   - Fecha: Selector de fecha estándar.
   - Booleano: Casilla de verificación (Sí/No).
   - Selección (Select): Permite definir una lista cerrada de opciones.
   - Relación (Relation): EL PARÁMETRO MÁS POTENTE. Permite conectar tablas.
     * Related Table: Tabla de donde provienen los datos.
     * Display Field: Campo que se mostrará en lugar del ID técnico.

2.2. FUNCIONALIDADES DEL CONSTRUCTOR:
   - Reordenamiento: Los campos pueden organizarse mediante un "Order Index".
   - Obligatoriedad: Parámetro "Required" para validar la entrada de datos.

--------------------------------------------------------------------------------
3. MÓDULO DATA MANAGER: GESTIÓN DE INFORMACIÓN
--------------------------------------------------------------------------------

3.1. PROCESOS DE DATOS:
   - Inserción Dinámica: El sistema genera formularios basados en la estructura
     del Builder en tiempo real.
   - Movimiento/Clonación: Permite "Mover" una tabla de un Sistema A a un Sistema B,
     clonando toda la estructura y los registros automáticamente.

3.2. SERVICIOS DE EXPORTACIÓN:
   - CSV: Descarga compatible con Excel y otras hojas de cálculo.
   - JSON: Formato estructurado para desarrolladores o integraciones externas.

--------------------------------------------------------------------------------
4. MÓDULO DATIUM AI: EL CEREBRO DEL SISTEMA
--------------------------------------------------------------------------------

La IA de Datium (GLM-4) no solo responde preguntas, sino que actúa sobre el sistema.

4.1. CAPACIDADES DEL CHAT:
   - Consultas de Datos Reales: "Muestra los 5 productos más caros".
   - Análisis Multimodal/Archivos: Soporta la subida de PDF, DOCX y TXT. La IA
     lee el contenido y lo integra en la conversación para análisis o carga.
   - Acciones Automatizadas: Puedes pedirle:
     "Crea un sistema llamado Ventas con una tabla Clientes que tenga nombre y email".
     La IA generará la estructura automáticamente tras tu confirmación.

4.2. POLÍTICA "CERO DUDAS":
   Datium AI está configurada para evitar alucinaciones. Si el dato existe en el
   sistema, lo devolverá literalmente. Si no, informará que la tabla está vacía.

--------------------------------------------------------------------------------
5. SERVICIOS GENERALES DE LA WEB APP
--------------------------------------------------------------------------------

5.1. GESTIÓN DE ALMACENAMIENTO (MEDIA SERVICE):
   - Los usuarios pueden subir imágenes para sus sistemas y avatares.
   - El sistema calcula el "Storage Used" en bytes para aplicar límites de plan.

5.2. PLANES Y CAPACIDADES (LIMITS):
   - Gratuito: Límites ajustados en sistemas y tablas.
   - Pro/Business: Aumenta los límites de registros (hasta 50,000 por tabla),
     campos (hasta 200) y almacenamiento (hasta 1GB o más).

5.3. PANEL DE ESTADÍSTICAS:
   - Gráfico de actividad de 7 días.
   - Contador global de sistemas, usuarios y registros.
   - Distribución de modos de seguridad.
